# Contributing to futuapi4go-demo

Thank you for your interest in contributing!

## How to Contribute

1. **Open an issue first** — for bug reports, feature requests, or SDK corrections.
2. **Fork the repo** and create a branch: `git checkout -b fix/my-fix`.
3. **Make your changes** — keep commits atomic and well-described.
4. **Test with a simulator or local OpenD** before submitting.
5. **Open a pull request** with a clear description.

## Coding Standards

- Run `go build ./...` and `go vet ./...` before submitting.
- Keep the demo focused — it should remain a single-file, menu-driven showcase.
- New API additions should mirror the style of existing demo functions.

## Code of Conduct

By participating, you agree to uphold our community standards. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).
